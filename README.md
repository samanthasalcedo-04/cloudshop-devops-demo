#CloudShop DevOps Demo
## Forma de trabajo del equipo
Los cambios deben proponerse mediante ramas (branches) y pull requests (solicitudes de cambios) para asegurar la calidad y revisión del código. 

* **Flujo principal:** Todo cambio debe realizarse en una rama dedicada antes de integrarse a la rama principal (`main`).
* **Colaboración externa:** Si una persona no tiene permisos directos sobre el repositorio principal, debe trabajar desde un fork (bifurcación) y enviar un pull request desde allí.
* **Revisión:** Todo pull request debe ser revisado y aprobado antes de ser integrado al proyecto.
